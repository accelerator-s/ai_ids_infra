# AI-IDS-Infrastructure

## 项目简介

AI-IDS-Infrastructure 是一个面向 HTTP 明文流量的轻量级网络入侵检测系统。系统通过实时抓包和离线流量包分析两种方式获取网络流量，对常见 Web 攻击和异常访问行为进行检测，并通过 WebUI 页面提供操作入口、告警展示和可视化统计。

系统针对常见的SQL注入、XSS攻击、命令注入、路径遍历、敏感文件探测、DDOS/高频访问、暴力破解、Web扫描器等8中常见的网络攻击行为进行探测，通过`规则匹配 + 行为检测 + 风险评分`识别明确攻击，并使用 AI 辅助研判评分未达到直接告警阈值的模糊请求。任务完成后，AI 还会对检测结果进行汇总分析并生成评测报告。

## 项目目标

1. 支持对指定 IP 或域名进行实时流量抓取和分析。
2. 支持读取或上传 pcap 流量包进行离线分析。
3. 从网络数据包中解析源 IP、目的 IP、端口、协议、HTTP 请求路径、参数、Header、User-Agent 等信息。
4. 建立常见网络攻击特征库，覆盖 SQL 注入、XSS、命令注入、路径遍历、敏感文件探测、DDoS / 高频访问、暴力破解和 Web 扫描器行为 8 类攻击。
5. 结合规则匹配和行为统计，对 HTTP 明文流量中的攻击行为进行检测和风险评分。
6. 接入 AI 大模型，对存在可疑特征、但风险评分未达到直接告警阈值的请求进行辅助研判。
7. 对分析任务的告警和统计结果进行 AI 评测，生成包含风险概述、主要威胁和处置建议的报告。
8. 提供 WebUI 操作界面，支持抓包任务配置、pcap 文件分析、告警查看、AI 评测报告和统计图表展示等功能。

## 核心检测范围

系统重点检测以下 8 类常见攻击和异常行为。

| 编号 | 检测类型 | 检测方式 | 示例特征 |
| --- | --- | --- | --- |
| 1 | SQL 注入 | HTTP 内容规则匹配 | `or 1=1`、`union select`、`sleep()` |
| 2 | XSS 跨站脚本 | HTTP 内容规则匹配 | `<script>`、`onerror=`、`javascript:` |
| 3 | 命令注入 | HTTP 内容规则匹配 | `;`、`&&`、`whoami`、`cat /etc/passwd` |
| 4 | 路径遍历 | URL / 参数规则匹配 | `../`、`..\\`、`/etc/passwd` |
| 5 | 敏感文件探测 | 请求路径检测 | `/.env`、`/.git/config`、`/backup.zip` |
| 6 | DDoS / 高频访问 | 流量行为统计 | 短时间内大量请求 |
| 7 | 暴力破解 | 登录行为统计 | 多次登录失败、频繁访问登录接口 |
| 8 | Web 扫描器行为 | User-Agent + 行为检测 | `sqlmap`、`nikto`、大量 404 |

## 运行模式

系统设计两种流量分析模式，分别对应不同的使用场景。

### 1. 实时抓包分析模式

实时抓包分析模式用于监听指定目标的 HTTP 明文流量。用户可以在 WebUI 中配置监听网卡、目标 IP 或目标域名，系统根据配置启动抓包任务，并对抓取到的数据包进行实时解析和检测。

当用户指定 IP 时，系统直接基于该 IP 过滤流量；当用户指定域名时，系统先将域名解析为 IP 地址，再对解析得到的 IP 进行流量过滤和分析。

实时抓包分析流程：

```text
用户输入 IP / 域名 / 网卡
        │
        ▼
域名解析为 IP 地址
        │
        ▼
生成抓包过滤条件
        │
        ▼
PyShark 抓取流量
        │
        ▼
解析 HTTP 明文请求
        │
        ▼
规则检测 + 行为检测 + 风险评分
        │
        ▼
高风险请求直接告警，模糊请求交由 AI 辅助研判
        │
        ▼
生成告警和 AI 评测报告，并在 WebUI 中展示
```

### 2. pcap 离线分析模式

pcap 离线分析模式用于对已有流量包进行检测。用户可以通过 WebUI 上传 pcap 文件，或指定本地 pcap 文件路径，系统读取文件中的数据包并按时间顺序进行解析和检测。

pcap 离线分析流程：

```text
用户上传 / 选择 pcap 文件
        │
        ▼
读取 pcap 数据包
        │
        ▼
解析 IP / TCP / HTTP 信息
        │
        ▼
提取 HTTP 请求路径、参数、Header、Body等信息
        │
        ▼
规则检测 + 行为检测 + 风险评分
        │
        ▼
高风险请求直接告警，模糊请求交由 AI 辅助研判
        │
        ▼
生成离线分析结果和 AI 评测报告
```

## 功能模块设计

### 1. WebUI 操作与可视化模块

WebUI 是系统的主要操作入口，用于完成抓包配置、pcap 分析、结果查看和数据可视化。

页面结构：

```text
总览：模块就绪度、告警统计、攻击类型 / 风险等级分布、高频来源 IP、最近告警
实时抓包：网卡选择、目标 IP / 域名配置、任务启停（等待抓包模块落地）
离线分析：pcap 上传、分析任务列表（等待分析模块落地）
告警中心：按攻击类型 / 风险等级 / 来源 IP 筛选，查看命中规则和检测原因
AI 报告：选择任务生成评测报告、查看历史报告、失败报告展示原因
系统配置：服务端口、大模型接入参数（OpenAI 兼容接口）
```

页面已全部完成，不使用模拟数据：依赖后端模块的操作在模块落地前
会根据后端返回的 501 状态显示"模块待实现"提示，模块实现后自动启用。

### 2. 后端 API 服务模块

后端 API 负责连接前端页面、抓包任务、检测模块和数据库。

主要功能：

```text
系统状态查询：汇报各模块就绪情况，模块文件落地后自动检测
运行配置读写：服务端口、大模型参数，保存在数据库 settings 表
大模型接入：模型列表拉取、连通测试（OpenAI 兼容协议）
任务与告警：创建、更新、筛选查询
统计汇总：攻击类型 / 风险等级分布、高频来源 IP、最近告警
AI 评测报告：汇总任务告警并调用大模型生成报告
待实现模块占位：抓包、离线分析接口统一返回 501
```

路由定义和请求示例统一维护在仓库根目录的 [API.md](API.md)。

### 3. 实时流量采集模块

实时流量采集模块负责从指定网卡抓取网络数据包，并支持根据 IP 或域名进行过滤。

主要功能：

```text
获取本机可用网卡列表
接收用户输入的目标 IP 或域名
将域名解析为 IP 地址
生成 BPF 抓包过滤表达式
调用 PyShark 进行实时抓包
将抓取到的数据包发送给解析模块
```

示例过滤条件：

```text
host 192.168.1.10 and tcp port 80
host 93.184.216.34 and tcp port 80
```

### 4. pcap 离线分析模块

pcap 离线分析模块负责读取已有流量包文件，并对其中的数据包进行逐条分析。

主要功能：

```text
接收用户上传的 pcap 文件
读取本地 pcap 文件
按时间顺序解析数据包
提取 HTTP 明文请求信息
将解析结果发送给检测模块
生成离线分析结果和统计报告
```

### 5. 数据包与 HTTP 协议解析模块

该模块负责将原始数据包转换为系统可检测的结构化 HTTP 请求数据。

主要解析内容：

```text
源 IP
目的 IP
源端口
目的端口
协议类型
请求方法
请求路径
Query 参数
Header 信息
User-Agent
请求体内容
响应状态码
时间戳
```

结构化请求示例：

```json
{
  "src_ip": "192.168.1.20",
  "dst_ip": "192.168.1.10",
  "src_port": 53421,
  "dst_port": 80,
  "protocol": "HTTP",
  "method": "GET",
  "path": "/login",
  "query": "username=admin' or '1'='1",
  "headers": {
    "User-Agent": "Mozilla/5.0"
  },
  "body": "",
  "timestamp": "2026-07-08 10:00:00"
}
```

### 6. 规则特征库模块

规则特征库用于保存不同攻击类型的检测规则，便于后续维护和扩展。

规则示例：

```json
{
  "type": "SQL Injection",
  "level": "high",
  "patterns": [
    "union\\s+select",
    "or\\s+1\\s*=\\s*1",
    "sleep\\s*\\(",
    "--"
  ]
}
```

### 7. 规则检测模块

规则检测模块负责将解析出的 HTTP 请求内容与规则库进行匹配。

主要功能：

```text
加载 JSON 规则库
对请求路径、参数、Header、Body 进行规则匹配
检测 SQL 注入、XSS、命令注入、路径遍历等内容型攻击
检测敏感文件探测和扫描器 User-Agent
输出命中的规则、攻击类型、风险等级和检测原因
```

### 8. 行为检测模块

行为检测模块负责发现单个数据包中不明显、但从访问行为上可以识别的异常。

主要功能：

```text
统计同一 IP 在时间窗口内的请求次数
检测 DDoS / 高频访问行为
统计登录接口访问和失败次数
检测暴力破解行为
统计同一 IP 访问的不同路径数量
检测 Web 扫描器批量探测行为
统计敏感路径访问次数
统计 404 响应数量
```

示例规则：

```text
同一 IP 在 60 秒内请求超过 100 次，标记为疑似高频访问。
同一 IP 在 5 分钟内登录失败超过 10 次，标记为疑似暴力破解。
同一 IP 在 1 分钟内访问超过 30 个不同路径，标记为疑似扫描行为。
```

### 9. 风险评分模块

风险评分模块负责综合规则检测和行为检测结果，给出最终风险等级。

风险等级设计：

| 等级 | 分数范围 | 处理方式 |
| --- | --- | --- |
| normal | 0 - 19 | 未发现明显异常，不触发 AI 研判 |
| low | 20 - 39 | 存在轻微异常，交由 AI 辅助研判 |
| medium | 40 - 69 | 疑似攻击行为，交由 AI 辅助研判 |
| high | 70 - 89 | 明确攻击特征，直接生成告警 |
| critical | 90 及以上 | 高频或严重攻击行为，直接生成高危告警 |

评分参考：

```text
命中高危规则：+70
命中中危规则：+40
命中低危规则：+20
短时间高频请求：+30
登录失败次数异常：+30
访问敏感路径：+30
扫描器 User-Agent 命中：+50
```

### 10. AI 辅助研判模块

AI 辅助研判模块用于处理已命中可疑特征、但风险评分还不足以直接告警的模糊请求。当风险分数在 20 至 69 之间时，系统将 HTTP 请求、命中规则、行为检测结果和风险分数一并交给 AI，由 AI 结合请求上下文给出辅助研判结果。

请求分数达到 70 分时直接告警，不再等待 AI 研判。低于 20 分的请求不调用 AI。这样既保证明确攻击能够及时告警，也能够进一步分析单凭规则难以确定的请求。

辅助研判流程：

```text
规则检测 + 行为检测
        │
        ▼
计算风险分数
        │
        ├── 70 分及以上：直接生成告警
        │
        ├── 20 - 69 分：交由 AI 辅助研判
        │                     │
        │                     ├── 判定为攻击：生成告警
        │                     └── 判定为正常：保留研判记录
        │
        └── 低于 20 分：不触发 AI 研判
```

AI 输出示例：

```json
{
  "ai_judgement": "malicious",
  "attack_type": "SQL Injection",
  "confidence": 0.87,
  "reason": "请求参数中的条件表达式与异常注释符组合后具有明显的 SQL 注入意图"
}
```

AI 调用超时、返回格式错误或无法完成研判时，系统保留原始风险分数和规则命中结果，并将该请求标记为待人工复核，不应默认按正常请求处理。

### 11. AI 评测报告模块

AI 评测报告模块负责对已完成的检测任务进行汇总分析。系统将任务摘要、攻击类型分布、风险等级分布、高频来源 IP 和典型告警整理为 AI 输入，由 AI 生成风险概述、主要威胁和处置建议。

评测报告与单条请求的辅助研判相互独立。报告模块只读取已保存的检测和研判结果，不再修改风险分数或告警状态。AI 报告调用失败时，已有检测结果和告警数据保持不变，并记录报告生成失败的原因。

报告生成流程：

```text
完成实时抓包或 pcap 分析任务
        │
        ▼
汇总任务信息、统计结果和典型告警
        │
        ▼
按照 Prompt 模板组装 AI 输入
        │
        ▼
AI 生成结构化评测报告
        │
        ▼
保存报告并在 WebUI 中展示
```

报告内容：

```text
任务基本信息和流量概况
告警总数与风险等级分布
主要攻击类型和高频来源 IP
典型告警及其命中规则
整体风险概述
需要优先关注的威胁
处置与防护建议
```

AI 输出示例：

```json
{
  "summary": "本次任务共分析 86 条 HTTP 请求，生成 12 条告警。",
  "risk_assessment": "告警主要为 SQL 注入和敏感文件探测，整体风险等级为 high。",
  "key_findings": [
    "192.168.1.20 在短时间内多次访问敏感路径",
    "登录接口出现多次 SQL 注入规则命中"
  ],
  "recommendations": [
    "核查高频来源 IP 的访问记录",
    "对登录接口增加参数化查询和访问频率限制"
  ]
}
```

### 12. 数据存储与告警模块

该模块负责保存检测过程中的告警、流量摘要、AI 辅助研判记录、统计结果和 AI 评测报告。

告警字段建议：

```text
告警 ID
源 IP
目标 IP
源端口
目标端口
请求方法
请求路径
攻击类型
风险等级
风险分数
命中规则
是否由 AI 辅助研判
AI 研判结果
AI 研判原因
检测原因
时间戳
处理状态
```

## 技术栈

| 模块 | 实现方式 |
| --- | --- |
| 后端服务 | Python FastAPI + Uvicorn |
| 抓包以及流量解析 | PyShark（抓包模块落地后使用） |
| 规则检测 | Python 正则表达式 + JSON 规则库 |
| 行为统计 | Python 时间窗口统计 |
| 数据存储 | SQLite + SQLAlchemy |
| 运行配置 | WebUI 配置面板 + settings 表（不使用 .env 等配置文件） |
| AI 辅助研判与评测报告 | OpenAI 兼容大模型 API + JSON 结构化输出 |
| 前端 | 原生 HTML + CSS + JavaScript（ES Modules 组件化，无构建步骤） |
| 可视化图表 | 自绘 CSS / SVG 图表 |
| 测试流量 | HTTP 明文 pcap 样本 |

## 系统环境依赖

1. Python 3.10 及以上，依赖见 `requirements.txt`。
2. 抓包和 pcap 解析使用 PyShark，依赖 tshark。这两个模块尚未实现，
   当前运行核心服务不需要安装；实现落地后需先安装 Wireshark 并确认
   `tshark` 命令可用：

```bash
tshark -v
```

## 系统运行方式

```bash
# 安装依赖
.venv/bin/pip install -r requirements.txt

# 启动服务（默认 127.0.0.1，端口读取数据库配置，初始为 8000）
python -m app.main

# 或者临时指定监听地址和端口
python -m app.main --host 0.0.0.0 --port 8080
```

启动后浏览器访问 `http://127.0.0.1:8000` 打开 WebUI。

所有运行配置（服务端口、大模型接入参数）都在 WebUI 的"系统配置"页
修改，保存在 SQLite 的 `settings` 表中，项目不使用 .env 或其他配置
文件。端口修改需要重启服务进程才会生效。
