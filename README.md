# AI-IDS-Infrastructure

## 项目简介

AI-IDS-Infrastructure 是一个面向 HTTP 明文流量的轻量级网络入侵检测系统。系统通过实时抓包和离线流量包分析两种方式获取网络流量，对常见 Web 攻击和异常访问行为进行检测，并通过 WebUI 页面提供操作入口、告警展示和可视化统计。

系统针对常见的SQL注入、XSS攻击、命令注入、路径遍历、敏感文件探测、DDOS/高频访问、暴力破解、Web扫描器等8中常见的网络攻击行为进行探测，通过`规则匹配 + 行为检测 + AI二次过滤`实现攻击行为告警。

## 项目目标

1. 支持对指定 IP 或域名进行实时流量抓取和分析。
2. 支持读取或上传 pcap 流量包进行离线分析。
3. 从网络数据包中解析源 IP、目的 IP、端口、协议、HTTP 请求路径、参数、Header、User-Agent 等信息。
4. 建立常见网络攻击特征库，覆盖 SQL 注入、XSS、命令注入、路径遍历、敏感文件探测、DDoS / 高频访问、暴力破解和 Web 扫描器行为 8 类攻击。
5. 结合规则匹配和行为统计，对 HTTP 明文流量中的攻击行为进行检测和风险评分。
6. 接入 AI 大模型，对疑似异常流量进行二次判断，辅助降低误报率。
7. 提供 WebUI 操作界面，支持抓包任务配置、pcap 文件分析、告警查看、统计图表展示等功能。

## 核心检测范围

系统重点检测以下 8 类常见攻击和异常行为。

| 编号 | 检测类型 | 检测方式 | 示例特征 |
| --- | --- | --- | --- |
| 1 | SQL 注入 | HTTP 内容规则匹配 | `or 1=1`、`union select`、`sleep()` |
| 2 | XSS 跨站脚本 | HTTP 内容规则匹配 | `<script>`、`onerror=`、`javascript:` |
| 3 | 命令注入 | HTTP 内容规则匹配 | `;`、`&&`、`whoami`、`cat /etc/passwd` |
| 4 | 路径遍历 | URL / 参数规则匹配 | `../`、`..\\`、`/etc/passwd` |
| 5 | 敏感文件探测 | 请求路径检测 | `/.env`、`/.git/config`、`/backup.zip` |
| 6 | DDoS / 高频访问 | 流量行为统计 | 短时间内大量请求 |
| 7 | 暴力破解 | 登录行为统计 | 多次登录失败、频繁访问登录接口 |
| 8 | Web 扫描器行为 | User-Agent + 行为检测 | `sqlmap`、`nikto`、大量 404 |

## 运行模式

系统设计两种流量分析模式，分别对应不同的使用场景。

### 1. 实时抓包分析模式

实时抓包分析模式用于监听指定目标的 HTTP 明文流量。用户可以在 WebUI 中配置监听网卡、目标 IP 或目标域名，系统根据配置启动抓包任务，并对抓取到的数据包进行实时解析和检测。

当用户指定 IP 时，系统直接基于该 IP 过滤流量；当用户指定域名时，系统先将域名解析为 IP 地址，再对解析得到的 IP 进行流量过滤和分析。

实时抓包分析流程：

```text
用户输入 IP / 域名 / 网卡
        │
        ▼
域名解析为 IP 地址
        │
        ▼
生成抓包过滤条件
        │
        ▼
PyShark 抓取流量
        │
        ▼
解析 HTTP 明文请求
        │
        ▼
规则检测 + 行为检测 + AI 二次过滤
        │
        ▼
生成告警并在 WebUI 中展示
```

### 2. pcap 离线分析模式

pcap 离线分析模式用于对已有流量包进行检测。用户可以通过 WebUI 上传 pcap 文件，或指定本地 pcap 文件路径，系统读取文件中的数据包并按时间顺序进行解析和检测。

pcap 离线分析流程：

```text
用户上传 / 选择 pcap 文件
        │
        ▼
读取 pcap 数据包
        │
        ▼
解析 IP / TCP / HTTP 信息
        │
        ▼
提取 HTTP 请求路径、参数、Header、Body等信息
        │
        ▼
规则检测 + 行为检测 + AI 二次过滤
        │
        ▼
生成离线分析结果和统计报告
```

```text

```


### 6. 规则特征库模块

规则特征库用于保存不同攻击类型的检测规则，便于后续维护和扩展。

规则示例：

```json
{
  "type": "SQL Injection",
  "level": "high",
  "patterns": [
    "union\\s+select",
    "or\\s+1\\s*=\\s*1",
    "sleep\\s*\\(",
    "--"
  ]
}
```

### 7. 规则检测模块

规则检测模块负责将解析出的 HTTP 请求内容与规则库进行匹配。

主要功能：

```text
加载 JSON 规则库
对请求路径、参数、Header、Body 进行规则匹配
检测 SQL 注入、XSS、命令注入、路径遍历等内容型攻击
检测敏感文件探测和扫描器 User-Agent
输出命中的规则、攻击类型、风险等级和检测原因
```

### 8. 行为检测模块

行为检测模块负责发现单个数据包中不明显、但从访问行为上可以识别的异常。

主要功能：

```text
统计同一 IP 在时间窗口内的请求次数
检测 DDoS / 高频访问行为
统计登录接口访问和失败次数
检测暴力破解行为
统计同一 IP 访问的不同路径数量
检测 Web 扫描器批量探测行为
统计敏感路径访问次数
统计 404 响应数量
```

示例规则：

**待完善**
```text
同一 IP 在 60 秒内请求超过 100 次，标记为疑似高频访问。
同一 IP 在 5 分钟内登录失败超过 10 次，标记为疑似暴力破解。
同一 IP 在 1 分钟内访问超过 30 个不同路径，标记为疑似扫描行为。
```

### 9. 风险评分模块

风险评分模块负责综合规则检测、行为检测和 AI 判断结果，给出最终风险等级。

风险等级设计：

| 等级 | 含义 |
| --- | --- |
| low | 存在轻微异常，需要记录 |
| medium | 疑似攻击行为，需要关注 |
| high | 明确攻击特征，需要告警 |
| critical | 高频攻击或严重攻击行为，需要重点展示 |

### 10. AI 二次过滤模块

AI 二次过滤模块用于对疑似异常流量进行辅助判断，主要用于降低误报和识别变形攻击。

适合进入 AI 二次判断的情况：

**待完善**


## API 接口设计

### 1. 启动实时抓包分析

```http
POST /api/capture/start
```

请求示例：

```json
{
  "interface": "eth0",
  "target_type": "domain",
  "target": "example.com",
  "port": 80
}
```

响应示例：

```json
{
  "task_id": "capture-001",
  "status": "running",
  "resolved_ips": ["93.184.216.34"],
  "filter": "host 93.184.216.34 and tcp port 80"
}
```

### 2. 停止实时抓包分析

```http
POST /api/capture/stop
```

请求示例：

```json
{
  "task_id": "capture-001"
}
```

### 3. 上传 pcap 文件并分析

```http
POST /api/pcap/analyze
```

请求内容：

```text
multipart/form-data 上传 .pcap 文件
```

响应示例：

```json
{
  "task_id": "pcap-001",
  "status": "completed",
  "total_packets": 1200,
  "http_requests": 86,
  "alerts": 12
}
```

### 4. 查询告警列表

```http
GET /api/alerts
```

### 5. 查询告警详情

```http
GET /api/alerts/{alert_id}
```

### 6. 查询统计信息

```http
GET /api/stats
```

统计信息包括：

```text
告警总数
攻击类型分布
风险等级分布
高频来源 IP
最近告警记录
实时抓包任务状态
pcap 分析任务状态
```

### 7. 清空开发测试数据

```http
POST /api/dev/reset-database
```

该接口用于清空 `tasks` 和 `alerts` 表中的数据，并保留数据库文件和表结构不变。为了避免误操作，请求体中必须显式传入 `confirm: true`。

请求示例：

```json
{
  "confirm": true
}
```

响应示例：

```json
{
  "status": "reset",
  "deleted_alerts": 10,
  "deleted_tasks": 3
}
```
